Step 1 · Agent Wallet

The customer’s bank-app login — identity and spending authority established before the first HTTP request.

A Coinbase Smart Wallet on Base. The agent holds keys via passkey (WebAuthn), delegates spending authority through EIP-1193, and signs EIP-3009 authorizations for gasless USDC transfers. Identity and funding are established at L4 Account and L5 Application — both above the enforcement line.

Step 2 · HTTP 402 Payment Gate

The locked door — the server returns 402 Payment Required with machine-readable payment terms in the header.

The agent hits the merchant endpoint. Without a payment header, the Cloudflare Worker returns 402 Payment Required — the HTTP status code that has waited 25 years for this use case. The PAYMENT-REQUIRED header contains base64-encoded JSON: scheme, network, asset, amount, payTo address, and resource identifier.

This is a Gate checkpoint at L3 Execution, exactly on the enforcement line. The payment IS the authentication — no API key, no OAuth, no account. The gate lives at the boundary between policy and code.

// Hit the merchant gate — no payment header yet const gate = await fetch( 'https://stablekya.com/api/report' ); // Decode the payment requirements const header = gate.headers.get('PAYMENT-REQUIRED'); const requirements = JSON.parse( atob(header) ).accepts[0]; console.log(gate.status); // 402 console.log(requirements.asset); // "USDC" console.log(requirements.amount);// "1000" (= $0.001)

Step 3 · Payment & Delivery

The atomic exchange — signed payment, OFAC screening, on-chain settlement, and resource delivery in a single retry of the original request.

The agent signs an EIP-3009 transferWithAuthorization for 1000 units ($0.001 USDC), attaches the signature to the original request, and retries. The Coinbase CDP x402 facilitator screens the transfer against Chainalysis OFAC (code-enforced gate at L3), submits to Base, and returns the resource on 200 OK.

Three layers light simultaneously: L5 Application (the retry), L4 Account (the signed authorization), and L3 Execution (OFAC screening + on-chain settlement). The bracket spans the enforcement line — policy and code work together.
